**2명이 16주 동안, “설계 먼저 함께 → 나눠서 구현”**으로, GitHub·이력서 기반 **자소서 도우미 서비스**를 만든다.

## 전체 흐름 (4단계)

| 단계 | 주차 | 뭘 하나 | 쉽게 말하면 |
|------|------|----------|-------------|
| **1. 설계** | 1~2주 | 둘 다 함께 **서비스 설계** (어떤 기능·화면·데이터 구조로 갈지 문서로 정리) | 설계만 하고, 아직 역할 안 나눔. 설계 확정 후 3주부터 나눔. |
| **2. 기반 구축** | 3~8주 | **A**: 이력서·합격 자소서 DB, GitHub 코드 수집·정리  
**B**: 로그인( GitHub 연동), 채용 공고 분석, “나와 이 공고 맞나?” 점수, 문항별 전략 | 데이터 쌓고, 로그인·공고 분석·전략까지 준비. |
| **3. 자소서 AI** | 9~11주 | **B**: AI가 **자소서 초안** 짜 주기, 사용자가 고치면 AI가 **다시 첨삭**해 주기  
**A**: 초안/첨삭에 필요한 데이터 제공·검색 기능 | “자소서 써줘” → 초안 → 수정 → “다시 봐줘” 반복. |
| **4. 통합·마무리** | 12~16주 | 둘 다: 전체 흐름 연결, 버그 수정, 테스트, 데모 준비 | 처음부터 끝까지 한 번에 써보고, 16주 차에 데모. |

---

## 역할 나누기 (3주부터)

| 담당 | 쉽게 말하면 맡는 일 |
|------|---------------------|
| **A (데이터)** | 이력서·합격 자소서 저장·검색, GitHub 코드 수집·요약·성과 문장 뽑기, 그걸 쓰는 API(서버 기능) 만들기 |
| **B (서비스·AI)** | GitHub 로그인, 채용 공고 분석, “나–공고 맞춤도” 점수, 문항별 전략, **자소서 초안 생성 AI**, **첨삭 AI** 흐름 설계·구현 |

→ **설계(1~2주)**는 A/B 구분 없이 **둘 다 함께** 진행.

---

## 16주가 끝나면 나오는 것

- GitHub로 로그인해서 **내 레포만** 골라 분석
- **채용 공고 URL** 넣으면 업무·역량·인재상 추출 + “나와 맞는지” 점수
- **자소서 문항** 넣으면 문항 유형·쓸 만한 경험(에셋)·부족한 점(Gap) 정리
- **AI가 자소서 초안** 작성 → 사용자가 수정 → **AI가 다시 첨삭** (반복 가능)
- 합격 자소서 검색·참고
