# Autofolio 채용 공고 파싱 전략

**문서 버전:** 1.1  
**최종 정리일:** 2025-02-12

---

## 1. 개요

Phase 2 (Strategy Intelligence)에서 채용 공고 URL을 입력받아 **LLM이 직접 웹에 접근하여 구조화된 정보를 추출**하는 전략.

### 핵심 컨셉

> **"크롤링과 추출을 분리하지 않고, LLM이 한 번에 처리"**

기존 방식 (크롤링 → 전처리 → LLM)이 아닌, **LLM + Web Browsing**으로 URL 접근부터 추출까지 단일 스텝으로 처리.

### 목표

채용 공고에서 다음 3가지 핵심 정보를 추출:

1. **주요 업무** (Key Responsibilities)
2. **핵심 역량** (Requirements / Skills)
3. **기업 인재상** (Company Values / Culture Fit)

---

## 2. 파이프라인

```
[유저] 채용 공고 URL 입력
         │
         ▼
┌─────────────────────────────────────┐
│ LLM + Web Browsing (단일 스텝)        │
│                                     │
│ 1. URL 접근                         │
│ 2. 페이지 내용 읽기                  │
│ 3. 구조화 추출 (주요 업무/역량/인재상)  │
│                                     │
│ → 크롤링/전처리/추출 분리 없이 한 번에  │
└────────────────┬────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────┐
│ 구조화된 JSON 출력                   │
│ - 주요_업무                         │
│ - 핵심_역량                         │
│ - 기업_인재상                        │
└─────────────────────────────────────┘
```

---

## 3. LLM 기반 크롤링의 장점

| 항목 | 설명 |
|------|------|
| **파이프라인 단순화** | 크롤링 → 전처리 → LLM 분리 불필요. 한 번에 끝. |
| **유지보수 감소** | 사이트별 HTML 파싱 로직 필요 없음 |
| **유연함** | LLM이 알아서 페이지 구조 파악, 다양한 포맷 대응 |
| **비정형 처리** | nav/footer/광고 등 노이즈를 LLM이 자동 무시 |

---

## 4. 구현 옵션

| 방식 | 설명 | 비고 |
|------|------|------|
| **GPT-4 + Web Browsing** | OpenAI API에서 browsing tool 활용 | ChatGPT Plus / API |
| **Claude + MCP Tool** | MCP로 웹 fetch 기능 연결 후 LLM이 처리 | Tool Use 기반 |
| **LangChain + WebBaseLoader** | URL 로드 → LLM 추출 체인 | 간단한 구현 |
| **Browser-Use Agent** | LLM이 브라우저 직접 제어 | 동적 페이지 대응 |

### 권장

- **1차:** LangChain WebBaseLoader 또는 Claude MCP Tool로 간단히 구현
- **폴백:** 접근 실패 시 유저가 텍스트 직접 붙여넣기

## 5. 프롬프트 전략

### 5.1 추출 항목 정의

| 항목 | 설명 | 예시 |
|------|------|------|
| **주요 업무** | 포지션에서 수행할 핵심 업무/책임 | "백엔드 API 설계 및 개발", "데이터 파이프라인 구축" |
| **핵심 역량** | 필수/우대 기술 스택, 경력, 자격 요건 | "Python 3년 이상", "AWS 경험", "CS 전공" |
| **기업 인재상** | 회사가 원하는 인재상, 문화 fit, 소프트 스킬 | "주도적", "성장 지향", "커뮤니케이션 능력" |

### 5.2 프롬프트 템플릿

```
당신은 채용 공고 분석 전문가입니다.

다음 URL의 채용 공고 페이지에 접근해서, 아래 3가지 항목을 JSON으로 추출해주세요.

## URL
{job_posting_url}

## 추출 항목

1. **주요_업무**: 이 포지션에서 수행할 핵심 업무와 책임 (리스트)
   - 구체적인 업무 내용 위주로 추출
   - 너무 일반적인 표현("업무 수행" 등)은 제외

2. **핵심_역량**: 필수 및 우대 요건 (리스트)
   - 기술 스택 (언어, 프레임워크, 툴)
   - 경력 요건 (연차, 도메인 경험)
   - 자격 요건 (학력, 자격증 등)
   - 우대사항은 "(우대)" 표시

3. **기업_인재상**: 회사가 원하는 인재의 특성 (리스트)
   - 소프트 스킬 (커뮤니케이션, 협업 등)
   - 가치관 (주도성, 성장 마인드 등)
   - 문화 fit 관련 키워드

4. **회사명**: 채용 공고의 회사명
5. **포지션명**: 채용 포지션 제목

## 출력 형식 (JSON)

{
  "회사명": "...",
  "포지션명": "...",
  "주요_업무": ["...", "..."],
  "핵심_역량": ["...", "..."],
  "기업_인재상": ["...", "..."]
}
```

### 5.3 출력 예시

**입력:** `https://example.com/jobs/backend-developer`

**출력:**
```json
{
  "회사명": "테크스타트업",
  "포지션명": "백엔드 개발자",
  "주요_업무": [
    "REST API 설계 및 개발",
    "대용량 트래픽 처리를 위한 시스템 최적화",
    "데이터베이스 설계 및 쿼리 최적화"
  ],
  "핵심_역량": [
    "Python 또는 Java 능숙",
    "백엔드 개발 경력 3년 이상",
    "RDBMS 및 NoSQL 사용 경험",
    "AWS/GCP 클라우드 경험 (우대)",
    "대규모 트래픽 처리 경험 (우대)",
    "오픈소스 기여 경험 (우대)"
  ],
  "기업_인재상": [
    "문제를 끝까지 파고드는 집요함",
    "동료와 적극적으로 소통",
    "새로운 기술에 대한 호기심"
  ]
}
```

## 6. 활용 (Phase 2 연계)

| 추출 항목 | 활용 | 연계 |
|-----------|------|------|
| **주요 업무** | 유저 프로젝트 경험과 **직접 매칭** | 폴더 요약(Mid-level)과 유사도 검색 → 메인/보조 무기 선정 |
| **핵심 역량** | 유저 기술 스택과 **스킬 매칭** | 파일/폴더 메타(language, tech)와 비교 → Gap 분석 |
| **기업 인재상** | 자소서 **톤/강조점** 가이드 | Writer 프롬프트에 반영 → "주도성 강조", "협업 사례 포함" 등 |

---

## 7. 예외 처리

| 케이스 | 대응 |
|--------|------|
| **URL 접근 실패 (로그인 필요, 차단 등)** | 유저에게 텍스트 직접 붙여넣기 옵션 제공 |
| **공고 내용이 너무 짧음** | 최소 항목만 추출, 부족 시 유저에게 보완 요청 |
| **비정형 포맷 (이미지 기반 등)** | 직접 입력 안내 (또는 추후 OCR 확장) |
| **외국어 공고** | LLM이 다국어 지원하므로 그대로 처리, 출력은 한국어로 통일 |

## 9. 요약

| 항목 | 내용 |
|------|------|
| **입력** | 채용 공고 URL (또는 텍스트 직접 입력) |
| **방식** | **LLM 기반 크롤링** (웹 접근 + 추출 단일 스텝) |
| **추출 항목** | 주요 업무, 핵심 역량, 기업 인재상 (+회사명, 포지션명) |
| **출력** | JSON 형태의 구조화된 공고 정보 |
| **활용** | 유저 에셋과 매칭 → 메인/보조 무기, Gap 분석, Writer 톤 가이드 |
| **폴백** | URL 접근 실패 시 텍스트 직접 입력 |
