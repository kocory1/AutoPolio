# Autofolio RAG 파이프라인 핵심 (순서)

**Retrieval-Augmented Generation** 흐름만, 순서대로 요약.

---

## 1. 인덱싱 (무엇을 저장·검색 가능하게 할지)

| 순서 | 단계 | 설명 |
|------|------|------|
| 1 | **User 프로필** | GitHub 코드 + 이력서/포트폴리오 → 전처리 → **RAPTOR 임베딩**(파일→폴더 요약→루트) → **Vector DB** 저장 |
| 2 | **에셋** | 코드·커밋·문서 → STAR 성과 문장 후보 생성 → User 프로필과 함께 관리 |
| 3 | **합격 자소서** | 잡코리아/링커리어 크롤링 → DB 저장 (문항/회사/연도·키워드로 검색 가능) |

→ 여기까지가 **Retrieval할 수 있는 데이터**를 만드는 단계.

---

## 2. 검색(Retrieval) — 언제 무엇을 가져오나

| 순서 | 시점 | 무엇을 검색 | 어디에 쓰나 |
|------|------|-------------|-------------|
| 1 | **Job Fit** | 채용 공고 임베딩 vs **User 프로필 임베딩** 유사도 | "나–공고 맞춤도" 점수·순위 |
| 2 | **전략 수립** | 문항·공고에 맞는 **에셋**(User 프로필 기반) 조회 | 문항별 "쓸 무기" 선정, Gap 분석 |
| 3 | **Writer** | **합격 자소서** 검색(문항/회사/키워드) | 초안 작성 시 참고 샘플(retrieve_samples) |
| 4 | **Inspector** | (필요 시) 초안·전략에 맞는 에셋/합격 샘플 재조회 | 첨삭 제안 시 근거 보강 |

---

## 3. 생성(Generation) — Retrieval 결과를 넣고 뭘 만드나

| 순서 | 단계 | 입력(Retrieval 포함) | 출력 |
|------|------|----------------------|------|
| 1 | **전략 수립** | 문항 + 공고(파싱) + **에셋(조회)** | 문항 유형·매칭 에셋·Gap·전략 JSON |
| 2 | **Writer** | 전략 + 에셋 + **합격 자소서(검색)** | 자소서 **초안** |
| 3 | **Inspector** | 초안 + 전략 + 에셋 (필요 시 재검색) | **보완 제안** (Human 수정 후 반복 가능) |

---

## 한 줄로 보는 RAG 흐름

```
[인덱싱] User 프로필(RAPTOR) + 에셋 + 합격 자소서 → DB
    ↓
[검색]   Job Fit(공고↔프로필) / 전략(에셋) / Writer(합격 자소서) / Inspector(에셋·샘플)
    ↓
[생성]   전략 수립 LLM → Writer LLM(초안) → Inspector LLM(첨삭) → Human-in-the-loop
```
