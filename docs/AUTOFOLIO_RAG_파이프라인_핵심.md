# Autofolio RAG 파이프라인 핵심 (순서)

**문서 버전:** 1.0  
**기준:** [AUTOFOLIO_파이프라인.md](AUTOFOLIO_파이프라인.md), [AUTOFOLIO_임베딩전략.md](AUTOFOLIO_임베딩전략.md)

---

## 1. 개요

**Retrieval-Augmented Generation** 흐름만, 순서대로 요약. 인덱싱 → 검색 → 생성 단계로 정리.

---

## 2. 인덱싱 (무엇을 저장·검색 가능하게 할지)

| 순서 | 단계 | 설명 |
|------|------|------|
| 1 | **User 프로필** | GitHub 코드 + 이력서/포트폴리오 → 전처리 → **RAPTOR 임베딩**(파일→폴더 요약→루트) → **Vector DB** 저장 |
| 2 | **에셋** | 코드·커밋·문서 → STAR 성과 문장 후보 생성 → User 프로필과 함께 관리 |
| 3 | **합격 자소서** | 잡코리아/링커리어 크롤링 → DB 저장 (문항/회사/연도·키워드로 검색 가능) |

→ 여기까지가 **Retrieval할 수 있는 데이터**를 만드는 단계.

---

## 3. 검색(Retrieval) — 언제 무엇을 가져오나

| 순서 | 시점 | 무엇을 검색 | 어디에 쓰나 |
|------|------|-------------|-------------|
| 1 | **Job Fit** | **User DB**(프로필·임베딩) vs **공고 파싱 API 반환값** 비교 → Job Fit API | "나–공고 맞춤도" 점수·순위 |
| 2 | **Writer 준비** | **합격 자소서** 검색 모듈 호출 → **에셋**(User DB) 조회 (문항·공고 기반) | 초안 작성 입력 컨텍스트 구성. 순서: 유사 샘플 → 유저 DB |
| 3 | **Inspector** | **무조건** 에셋·합격 샘플 재조회 | 첨삭 제안 시 근거 보강 |

---

## 4. 생성(Generation) — Retrieval 결과를 넣고 뭘 만드나

| 순서 | 단계 | 입력(Retrieval 포함) | 출력 |
|------|------|----------------------|------|
| 1 | **Writer** | 문항 + 공고(파싱) + **합격 자소서(검색)** + **에셋(유저 DB 조회)** | 자소서 **초안** |
| 2 | **Inspector** | 초안 + 에셋·합격 샘플 (무조건 재조회) | **보완 제안** (Human 수정 후 반복 가능) |

---

## 5. 한 줄로 보는 RAG 흐름

```
[인덱싱] User 프로필(RAPTOR) + 에셋 + 합격 자소서 → DB
    ↓
[검색]   Job Fit(공고↔프로필) / Writer 준비(합격 자소서 → 에셋) / Inspector(에셋·샘플)
    ↓
[생성]   Writer LLM(초안) → Inspector LLM(첨삭) → Human-in-the-loop
```
